Step 5Ô∏è‚É£: Generate SSH Key Pair

On Your Local Machine (Laptop/Desktop):

Open Terminal and run:
ssh-keygen -t ed25519 -f ~/.ssh/my_gpu_cloud_key
Press Enter when prompted (no passphrase needed for simplicity)
Set correct permissions on the private key:
chmod 600 ~/.ssh/my_gpu_cloud_key
üìÅ This Creates Two Files:
  • ~/.ssh/my_gpu_cloud_key - Private key (keep secret!)
  • ~/.ssh/my_gpu_cloud_key.pub - Public key (safe to share)

Step 8Ô∏è‚É£: Upload Your Data

Transfer Files from Your Laptop to VM

Open a NEW terminal on your laptop (don't close the SSH session):
rsync -avz -P --checksum \
-e "ssh -i ~/.ssh/my_gpu_cloud_key" \
./my_data/ ubuntu@<VM_IP>:~/input_data/
Example:
rsync -avz -P --checksum \
-e "ssh -i ~/.ssh/my_gpu_cloud_key" \
./my_dataset/ ubuntu@192.168.1.100:~/input_data/
üìÇ What This Does:
  • Uploads ./my_data/ from your laptop
  • To ~/input_data/ in the VM's home directory
  • Shows progress with -P flag

üîß Troubleshooting Common Issues

Problem: "Permission denied (publickey)"

Solution:
  • Verify you imported the correct public key (.pub file)
  • Check the private key path in your SSH command
  • Ensure private key has correct permissions: chmod 600 ~/.ssh/my_gpu_cloud_key

Problem: rsync command not found

Solution:
  • Windows: Install rsync via WSL or use scp instead
  • Mac/Linux: rsync is pre-installed

Problem: Module command not found

Solution: Make sure you selected a base image with pre-installed modules (ml-dl-gpu-base-image or minimal-research-base-image)
RCCG @ HBKU

üîÑ Alternative: Using SCP Instead of rsync

If rsync is Not Available:

Upload files with scp:
scp -i ~/.ssh/my_gpu_cloud_key \
-r ./my_data/ \
ubuntu@<VM_IP>:~/input_data/
Download results with scp:
scp -i ~/.ssh/my_gpu_cloud_key \
-r ubuntu@<VM_IP>:~/output_data/ \
./my_results/
üìù Note: The -r flag means "recursive" - it copies entire directories including subdirectories.

üíæ Understanding Storage in This Scenario

Where Your Files Are Stored:

Root Disk (Ubuntu Home Directory)
  • Location: /home/ubuntu/ (or ~/)
  • Size: 100GB included with VM
  • Perfect for: Small input files, scripts, and results
  • Persistence: Data stays as long as VM exists
‚ö†Ô∏è Important Limitation:
  • Root disk data is deleted when you delete the VM
  • For this beginner scenario: Always download results before deleting VM
  • For larger projects: Use Volumes (covered in advanced tutorials)

üéÆ When Do You Need a GPU?

Use CPU-Only VM For:

  • Data preprocessing and cleaning
  • Testing code and scripts
  • Small statistical analyses
  • File organization and management

Use GPU VM For:

  • Deep learning model training
  • Large-scale neural network inference
  • Computer vision tasks (image processing)
  • Natural language processing (NLP)
  • Scientific simulations requiring parallel computing
üí° Pro Tip: Start with CPU VM for testing, then create a GPU VM only when you're ready for the actual computation. This saves significant costs!
